  🧮  GitHub Actions cron 최소 주기 — 5분이 한계

  ▎ "1분마다는 안 되나?" → GitHub Actions에서는 안 됩니다. 최소 5분.

  공식 제약

  GitHub 공식 문서 기준:

  ▎ "The shortest interval you can run scheduled workflows is once every 5 minutes."

  # ✅ 가능 (5분 간격)
  - cron: '*/5 * * * *'

  # ❌ 거부됨 (1분 간격)
  - cron: '*/1 * * * *'

  왜 5분인가?

  1. Free tier 자원 보호 — 1분 cron이면 하루 1,440회 × 모든 사용자 → 인프라 폭발
  2. Scheduler 스케일 — GitHub 전체 cron을 한 시스템에서 관리하기 위해
  3. 5분도 보장 X — 5–15분 jitter 정상. 정각(0 * * * *)은 가장 혼잡해서 더 늦어짐

  다른 시스템 비교

  시스템: GitHub Actions
  최소 주기: 5 min
  비용: 무료(public)
  비고: 5–15분 jitter, 정각 회피 권장
  ────────────────────────────────────────
  시스템: Linux cron (서버)
  최소 주기: 1 min
  비용: 서버비
  비고: 본인 서버 켜둬야 함 (= 우리가 피한 것)
  ────────────────────────────────────────
  시스템: AWS EventBridge / Lambda
  최소 주기: 1 min
  비용: 사실상 무료
  비고: AWS 계정 필요, 약간 셋업
  ────────────────────────────────────────
  시스템: Vercel Cron Jobs
  최소 주기: 1 min (Pro) / 1일 1회 (Hobby)
  비용: Pro $20/월부터
  비고: 무료는 1일 1회만
  ────────────────────────────────────────
  시스템: Cloudflare Workers Cron Triggers
  최소 주기: 1 min
  비용: 거의 무료
  비고: 100k 무료/일
  ────────────────────────────────────────
  시스템: Google Cloud Scheduler
  최소 주기: 1 min
  비용: 월 3개 무료
  비고: GCP 계정 필요
  ────────────────────────────────────────
  시스템: Render / Railway / Fly.io cron
  최소 주기: 1 min
  비용: 유료/일부 무료
  비고: PaaS 셋업
